# AI 위임/반려 로그

AI에게 위임한 작업, 받아들인 것과 반려/수정한 것, 그 과정에서 얻은 교훈을 매일 기록한다.

## 2026-07-20

**위임한 작업**: Phaser 3 + TypeScript + Vite 뱀서라이크 프로토타입 스파이크
- 플레이어 이동/카메라 추적, 0.5초 자동 공격, 적 스폰(동시 200마리, 오브젝트 풀링), 경험치 젬 드랍/흡수, FPS·적 수 HUD, 수치는 `src/data/`로 분리, 도형 아트만 사용, 물리 엔진 대신 거리 기반 충돌.

**반려/이슈**
- `npm create vite@latest` 가 이 셸(비대화형 stdin)에서 프롬프트 취소로 계속 실패함. `CI=true`, `--force` 옵션도 우회 못 함 → AI가 `package.json`/`tsconfig.json`/`vite.config.ts`를 수동으로 직접 작성해 스캐폴딩. 앞으로 비슷한 환경에서는 처음부터 수동 스캐폴딩을 기본값으로 고려.
- 전투(접촉 데미지 + 무적 프레임)는 첫 스파이크 단계에서 이미 구현되어 있었는데, 화면에 HP바가 없어서 실제로 체력이 깎이고 있는지 눈으로 확인할 방법이 없었음 → "로직이 맞게 짜여 있다"와 "화면에서 확인 가능하다"는 별개라는 교훈. 이후 상태값(체력 등)이 바뀌는 로직을 추가할 때는 반드시 눈에 보이는 UI(바/텍스트)를 같이 요구할 것.
- 이 세션의 브라우저 자동화 도구(chromium-cli 등) 부재로 AI가 직접 스크린샷으로 검증하지 못함. `tsc`/`vite build`/dev 서버 curl 확인까지만 AI가 하고, 실제 플레이 검증은 사람이 브라우저에서 직접 수행.

**받아들인 것**: 전투 로직, 오브젝트 풀 구조(`ObjectPool<T>`), 데이터 분리 패턴(`src/data/*Config.ts`) 그대로 채택.
